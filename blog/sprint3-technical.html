HTML, CSS, JavaScript and the DOM

Explaining HTML, CSS, JavaScript and the DOM

The Difference Between HTML and CSS

Creating a website – one that appeals to users both structurally and visually – is possible through the use of two languages, HTML and CSS. Together, these languages work together to dictate what is on a webpage and how it is displayed to a user.

HTML is used to deliver the actual content of a website. The components of this file will include your text elements, your sections, your images, your links and other information you want on a page. To describe HTML, people often use a house analogy, with this language likened to foundations, walls, rooms and the roof of the building.

CSS, on the other hand, is a language that gives a webpage its styling. With CSS, a developer can define the properties of HTML components to make a page look how they want it to. Colour, size, placement and many other characteristics of a website are created in this language. To continue with the house analogy, if HTML is the skeleton of a house, CSS is the decorative features; the carpet, the furniture, the paint and the room sizes.

Control Flow and Loops

In programming, control flow is the order that code is carried out by a computer. In JavaScript there are several ways that this order can be dictated by a developer. This will have an impact on the actual output of code and what the user experiences on a website.

One of these methods is a loop statement. This command tells a computer to continue performing a certain action until reaching a point specified by the developer. For example, a program might tell a computer to express a statement for a certain number of times – this probably wouldn’t be very useful but is just a simple example of what a loop could be used for.

Real-life situation examples of control flow and looping can make it easier to understand the concept. For example, say you had a bag full of $1 coins and you went to the plant store to buy a cactus. The cactus costs $50, so you need to sort through the bag and pull out 50 coins until you have enough to pay for your plant. If this situation was programmed, control flow would tell you when to start counting and when to hand over your money to their sales assistant so that it was the right amount to pay them. Looping would incur while you were pulling out the coins and adding them together in your hand, until you had enough.

The DOM

The DOM (Document Object Model) is the “the data representation of the objects that comprise the structure and content of a document on the web”, according to MDN. It is a depiction of a page that shows page structure, elements, style and content. The DOM is presented as a tree-like structure with branches of nodes representing components of a document.

A developer may interact with the DOM when applying JavaScript to manipulate components in the way they desire. Through the browser, it is possible to single out specific elements and classes and apply script to them so that they behave in certain ways.

Explain the difference between accessing data from arrays and objects.

Arrays and objects are similar in that they store data inserted in them by a developer. They are also alike in that at times, a developer may want to access the data stored in them.

The difference between an object and an array is that objects take in properties that are assigned values, while arrays are typically used to represent lists. To access data in an array, you can use the bracket method to return an item. By entering its index number(remember zero-based indexing), you can source its value from the array.

On the other hand, accessing data from an object is typically done by dot or bracket notation, along with the key of the value you want to return.

Explain what functions are and why they are useful

A lot of the time, developers will want to reuse the same snippets of code repeatedly. Rather than re-writing this each time, it’s possible to use a function to shorten code.

When the function is called, the computer will execute the code stored within the function!
